## What Is Pathfinder

Pathfinder is a local-data-first job search copilot for senior product leaders. It helps them find relevant roles sooner, prepare faster, and make better career decisions. The architecture (v3) is standalone HTML modules served via HTTP on a combined Node.js server (port 3000) that serves both static files AND the data persistence bridge API with zero npm dependencies.

## Canonical Schemas (v7.5)

### Normalization Rules
- `compensation`: always `{ min, max, currency, type, raw }` — never use `salary`
- `level`: single field — never use `targetLevel`
- `stage` (Company): canonical taxonomy — never use `fundingStage`
- `scoring`: `{ titleFit, domainFit, levelFit, companyFit, compensationFit, locationFit }` — same shape on roles and feed items. Each dimension 0-100. Weighted: title 25%, domain 20%, level 15%, company 15%, comp 15%, location 10%. Generated by `score-engine.js` using `processJD()` + user prefs.
- `reasons`: `string[]` — one reason per scoring dimension, set alongside `scoring` by score-engine.js
- `companyStage` (FeedItem): transient intake field, mapped to `stage` on Company Record on approval
- `lastActivity`: canonical timestamp for last update — never use `dateUpdated`

### Company Stage Taxonomy
`Public` | `Late-stage private` | `Growth-stage` | `Early-stage` | `Bootstrapped / Private` | `Unknown`

### Research Brief Sections
**Required (5):** snapshot, existence, plausible, screenOut, nextSteps
**Expandable (9):** pursuitEconomics, companyMarket, needs, fit, gaps, network, interview, proofPoints, dealBreaker

## Mandatory Rules

1. **PRD sync rule:** Every code change must update the relevant component PRD, CHANGELOG, and this file.
2. **Schema discipline:** Use canonical field names only. No legacy aliases.
3. **Privacy visibility:** Every generation action must indicate whether data leaves the local system.
4. **Resume provenance:** Every AI suggestion must trace to bullet bank, base resume, or locally stored story-bank content.
5. **Graceful degradation:** Core non-AI workflows must work when bridge is down.
6. **No deferred modules in nav:** Only Dashboard + 4 core modules reachable from product navigation.

### Architecture Changes (v3)

**Port Setup:**
- **Port 3000:** All services (static files, artifacts, citations, briefs, backup, JD fetch, embeddings, vectors)
- **Port 3847:** Optional, Claude Code stdio only
- **Port 8080:** V2 server (legacy, for reference)

**Server Consolidation (v3.7.0):** All MCP endpoints consolidated into server.cjs on port 3000. Artifacts, citations, briefs, backup/restore, JD fetch all served from one process.

**Embeddings (v3.8.0):** Local transformer embeddings via @xenova/transformers with sharp stubbed out. In-memory vector store for semantic search. Model: all-MiniLM-L6-v2 (384 dims). Lazy-loaded on first API call.

**Bridge Evolution:**
- `bridge/src/bridge-standalone.ts` — Failed (sharp native binary issues with @xenova/transformers)
- `bridge/src/bridge-lite.cjs` — Created as fallback but unused (zero-dep approach favored)
- `server.cjs` — NOW: full MCP infrastructure server (v2.0.0) replaces standalone bridge entirely

**Data Layer:**
- Changed bridge URL from hardcoded `localhost:3458` to `window.location.origin` for dynamic routing
- Added `pushToBridgeIfEmpty()` function: syncs localStorage → bridge on first load when bridge has no data
- Data directory moved from `~/.pathfinder/data/` to `.pathfinder-data/` (project-relative) for VM/Mac sharing

**Cache Busting:**
- All module HTML files now reference `data-layer.js?v=5.0.0` to prevent stale JS from being served

### Known Issues
- VM ports are NOT forwarded to Mac — only ports with Mac-native servers work (e.g., port 8080 with Python server in v2)
- Research Brief and Resume Tailor generation via Cowork sessions (user copies Cowork-ready prompt templates from app)
- LinkedIn connections recovered from bridge on pipeline init; if bridge is down, connections won't load until next bridge-available session
- Vector store is in-memory — resets on server restart. Roles must be re-indexed via POST /api/vectors/index-roles after restart.

## How to Start the Application

**Option 1: With Node.js available (recommended)**
```bash
cd ~/Projects/job-search-agents-v3
./start.sh
```
This starts the combined server on port 3000 (static + bridge API integrated) and the MCP Artifacts server on port 3847.

**Option 1b: MCP server only (for Claude Code stdio mode)**
```bash
node mcp-server/dist/index.js
```
Runs MCP server in stdio mode for Claude Code / Cowork integration (no HTTP).

**Option 2: Python fallback (legacy)**
```bash
python3 -m http.server 8080 --directory modules
```
This serves static files on port 8080 but has no bridge API (v2 approach).

**Access:**
- http://localhost:3000 (v3 — combined server)
- http://localhost:8080 (v2 — legacy, for reference)
